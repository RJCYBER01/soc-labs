Overview

This lab focuses on analyzing security logs to identify suspicious activity, correlate events, and determine whether an incident requires escalation.

Tools used: SIEM (Splunk / ELK), Sysmon, Windows Event Logs, Linux auth logs.

Scenario

A workstation triggered multiple failed login attempts followed by a successful login from an unusual IP address. Your task is to investigate the logs and determine whether this is brute-force activity or legitimate user behavior.

Investigation Steps

  • Reviewed Windows Event ID 4625 (failed logon attempts).
  • Correlated timestamps with Event ID 4624 (successful logon).
  • Checked source IP and geolocation for anomalies.
  • Analyzed Sysmon Event ID 3 (network connections) for unusual outbound traffic.
  • Reviewed user activity logs for privilege escalation attempts.

Findings

The pattern of repeated failed logins followed by a successful login from a foreign IP suggests a possible brute-force attack. No privilege escalation was detected, but lateral movement attempts were observed.

Conclusion

This activity should be escalated to Tier 2 for deeper investigation. Recommended actions include resetting the user’s password, blocking the suspicious IP, and reviewing additional logs for lateral movement.
